Appears In: Functional Tree of Area F2 Provide Safety and Emergency Facilities, DFD F2.1.11 Manage eCall Messages
This Function shall be capable of providing the following facilities:<br/><br/>(1) The ability to receive eCall messages being sent from Vehicles that are operating in the road network.<br/>(2) The ability to request other functionality to provide details of how to communicate with the most appropriate Public Safety Answering Point (PSAP) provider which can respond to and service the eCall message.<br/>(3) The ability to pass on the eCall message to the most appropriate PSAP once the details of how to connect to it have been provided.<br/>(4) The ability to keep the communication channel between the Host Vehicle and the PSAP open.<br/>(5) The ability to pass through eCall response messages from the PSAP to the Host Vehicle and to send reports of the status of the eCall message response to other functionality.<br/>(6) The ability to periodically communicate with all available PSAP's and to update their status depending on the availability that they each report.<br/>(7) If the connection with the Vehicle is lost the ability to request the previously used contact detains from the Manage PSAP Data Stores function.<br/>

Details:
(a) at initial start-up and periodically thereafter request a list of available PSAP's from the Manage PSAP Data Stores function;<br/>(b) when the response to the request in (a) is received, use the data objects that it contains to send a status request to every available PSAP;<br/>(c) as the responses to the requests in (b) are received, send updates of the status of each PSAP to the  Manage PSAP Data Stores function;<br/>(d) continuously monitor for receipt of a data flow containing the eCall message from functionality in the Provide Support for Host Vehicle Systems Functional Area;<br/>(e) when a data flow in (d) is received, check that all the required data objects are present and if not send an indication that the eCall message needs to be sent again back to the functionality in the Provide Support for Host Vehicle Systems Functional Area;<br/>(f) if all the required data objects are found to be present in (e) extract the Vehicle location plus details of how to contact the Host Vehicle and send them to the Manage PSAP Data Stores function;<br/>(g) continuously monitor for receipt of a data flow providing a response to the data flow sent in (f) above;<br/>(h) when the data flow in (g) is received, extract the data it contains and use if to send the eCall message to the PSAP specified in the data flow;<br/>(i) once (h) has been completed, send a data flow containing a "first acknowledgement" of the eCall message to the functionality in the Provide Support for Host Vehicle Systems Functional Area;<br/>(j) if the data flow received in (g) did not include details of an appropriate PSAP, send a data flow containing a PSAP unavailable indication to the functionality in the Provide Support for Host Vehicle Systems Functional Area;<br/>(k) following on from (h) pass any messages received from the PSAP through to the Provide Support for Host Vehicle Systems Functional Area for output in the Host Vehicle<br/>(l) if there is no evidence that the message (k) has been received, request contact details for the Host Vehicle from the Manage PSAP Data Stores function;<br/>(m) use the contact details provided in (l) to send the message received from the PSAP to the Host Vehicle;<br/>(n) if after a period of time no further messages are received in (k), send a "eCall message response finished" indication to the Manage PSAP Data Stores function.<br/>
