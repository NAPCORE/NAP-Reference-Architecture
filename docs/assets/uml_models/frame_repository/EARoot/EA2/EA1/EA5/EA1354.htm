Appears In: DFD F5.11 Provide Driver Monitoring and eCall, Functional Tree of Area F5 Provide Support for Host Vehicle Services
This Function shall be capable of providing the following facilities:<br/><br/>(1) The ability to continuously monitor the status of the Driver.<br/>(2) The ability to use data collected about the Driver to determine whether or not there is any impairment of the Driver due to any reason, which shall include, but not be limited to, fatigue, alcohol/drug abuse, sudden health problems, prolonged inattention, etc.<br/>(3) The ability to report the results of its determinations about the status of the Driver to other functionality in the System.<br/>

Details:
(a) continuously monitor for the receipt of the driver status data flow;<br/>(b) use the data flow in (a) to determine the current status of the Driver;<br/>(c) send the data in (b) to the Record Data for Operations function for storage in the Operational Data Store;<br/>(d) when required request the data previously stored in (c) from Data Store management function;<br/>(e) compare the current data from (b) with the data in (d), and against some internal "norm" in order to identify any impairment;<br/>(f) if an impairment is detected in (e) then determine if the driver has failed or not;<br/>(g) if in (f) the driver has failed, send the driver failed data flow to the Provide In-vehicle eCall Facilities function;<br/>(h) if in (f) it is just the driver's performance that has deteriorated, send the driver status data flow to the Alert Driver to Status function, to the Communicate with In-vehicle Systems function in the driver impaired data flow, to the Monitor Driver function in the Manage Freight and Fleet Functional Area in the driver status data flow and to the Classify Host Vehicle Driver Behaviour function in the driver status for behaviour classification data flow.<br/>
