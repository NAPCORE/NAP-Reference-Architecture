This Function shall be capable of providing the following facilities:<br/><br/>(1) The ability for inter-vehicle (V2V) communication including protocols for access the communication channel and for data integrity control.<br/>(2) The ability to avoid false data generation and broadcasting shall be an integral part of the communication facility.<br/>(3) The ability to manage the communications in a manner that does not overload the communications infrastructure.<br/>

Details:
(a) continuously monitor for data flows that have been sent from the other vehicle, or from other parts of the system in the Host Vehicle;<br/>(b) when any of the data flows from other parts of the system in the Host Vehicle are received in (a), analyse their contents and send then to the Other Vehicle in the appropriate output data flow;<br/>(c) when any of the data flows from the Other Vehicle are received in (a), analyse their contents;<br/>(d) if in (c) the contents are found to contain, XFCD, local visibility, road information, local road surface state, traffic queue ahead, other vehicle location, ghost driver warning or lane departure warning data, send it to other parts of the system in the Host Vehicle using the appropriately named data flow;<br/>(e) send all other data found in (c) in the instructions & warnings from other vehicles data flow to the Manage Vehicle Communications to Driver function for processing before being output to the Driver.<br/>
