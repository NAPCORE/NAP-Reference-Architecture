Appears In: DFD F6.7 Manage General Trip Preferences
This Data Store shall be used within the Provide Traveller Journey Assistance Functional Area (F6).  It shall contain information objects that provided the personalised data needed to support the Traveller during all of their trips.  The support is provided both during the trip planning, as well as during the trip execution.  The Store is also used to prevent cumbersome and error prone repeated input of the same information by the Traveller.  It serves as a "memory" to assist the Traveller during the entire trip, for all travel modes, during every phase of the trip.<br/><br/>The information objects in the Store shall contain the following data objects:<br/>(1) Traveller s ID (number),<br/>(2) Traveller driver licence ID (number or characters),<br/>(3) Traveller passport ID (number),<br/>(4) Traveller's preferred language (characters),<br/>(5) Traveller's disabilities (characters),<br/>(6) Other Traveller information relevant to trip planning (characters),<br/>(7) Traveller's payment information (characters and/or numbers),<br/>(8) Description of cargo to be carried on trip (characters),<br/>(9) Number of passengers expected in Vehicle (number),<br/>(10) Prime trip planning criterion (characters),<br/>(11) Default departure location (characters and/or numbers, e.g. GPS/Galileo data),<br/>(12) Default destination location (characters and/or numbers, e.g. GPS/Galileo data),<br/>(13) Level of detail desired of trip itinerary (characters),<br/>(14) Road toll payment method (characters),<br/>(15) Vehicle breakdown assistance information (characters),<br/>(16) Way of communication with Traveller while on-trip (characters + numbers),<br/>(17) Route guidance facilitation parameters (characters),<br/>(18) Preferred travel modes and classes (characters),<br/>(19) Preferred hotel(s)/chain(s) (characters),<br/>(20) Preferred car rental organisation(s) (characters).<br/><br/>The information object for each Traveller must be kept separate with no possibility of inadvertent access to the information object by either another Traveller, or some form of un-authorised access.<br/>
