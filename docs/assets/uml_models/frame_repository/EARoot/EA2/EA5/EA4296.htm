Appears In: DFD F2.1.11 Manage eCall Messages
This Data Store shall be used within the Provide Security and Emergency Facilities Functional Area.  It shall contain information objects that provide details of each eCall message from Vehicles that are active at any instance in time.<br/><br/>Each information object in the store shall contain the following data objects for each active eCall message:<br/>(1) identity (VIN) of Vehicle from which eCall message originated (character string - numbers and letters),<br/>(2) communication information for the Veihcle from which the eCall message originated (character string - numbers and letters),<br/>(3) time at which eCall message was received from the Vehicle (digits),<br/>(4) current status of eCall message handling (number)<br/>(5) name of PSAP to which eCall message was sent (character string),<br/>(6) identity of PSAP to which eCall was sent (number - optional),<br/>(7) communication information for the Public Safety Answering Point (PSAP) to which eCall was sent (character string - numbers and letters).<br/><br/>Note that the functionality that manages this Data Store does not enable the Communications Operator or any other entity to access these information objects.  This is to protect the identity and communications information of the Vehicles from which eCall messages are being processed.<br/>
