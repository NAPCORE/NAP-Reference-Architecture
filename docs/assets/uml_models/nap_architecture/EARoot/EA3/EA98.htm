The Functional View focuses on the functional aspects of a system. Defines the architectural elements that deliver the system’s functionality documenting the system’s functional structure including the key functional elements their responsibilities the interfaces they expose and the interactions between them. <br/>The View contains a chain of <b>Functions</b> connected via the <b>Functional data flows</b>, at both ends with a <b>Terminator.</b> This chain represents a process of data processing in a designed system. <b>Data stores</b> represent the ability of a function to store and retrieve data and model real databases. <br/>Functions are for clarity purposes grouped into modules - see physical view.<br/>The Functional View is elaborated with respect of minimum functionality of two NAP types: <b>Metadata Directory </b>and <b>Data Platform, </b>building upon the functionality of the previous type<br/>Those types are presented via TWO diagrams, each depicting functionality of respective type.<br/>
