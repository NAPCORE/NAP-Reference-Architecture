This Data Store shall be used within the Manage Traffic Functional Area.  It shall contain information objects that provide details of NAP Users, i.e. NAP Content Providers and NAP Content Consumers (for Data Platform NAP type) .  The Data Store contains private information used by NAP Users to access the NAP, to provide agreement on terms and conditions. Public metadata about traffic and travel information and other related information are stored in NAP Metadata store (D3.15). <br/><br/>For Content Providers the data store contains following data objects: <br/>(1) organization with name (1.1), postal address (1.2), website (1.3), agreement on licence terms (1.4), unique identifier assigned to organization (1.5), name and contact details of a responsible person to act as a contact (1.6). <br/>(2) user account(s) with name (2.1), email (2.2), credentials (2.3), user type (2.4). <br/>*(3) machine account with url (3.1), connection type (3.2), credentials (3.3). <br/><br/>For Content Consumers the data store contains following data objects: <br/>*(1) organization with name (1.1), postal address (1.2), website (1.3), agreement on licence terms (1.4), unique identifier assigned to organization (1.5), name and contact details of a responsible person to act as a contact (1.6). <br/>*(2) user account(s) with name (2.1), email (2.2), credentials (2.3), user type (2.4). <br/>*(3) machine account with url (3.1), connection type (3.2), credentials (3.3). <br/>* Optional data object for Data Platform NAP type and even there it is optional provided the data is not openly accessible.  <br/><br/>Additionally, for both: <br/>(4) Event information (log in, log out, change of data) <br/><br/> ========== <br/><br/>TODO: <br/><br/>- check what is in data flows and if it is covered by the DS <br/>- IMPORTANT: decide how to deal with optional nap.cc logging in. <br/>
