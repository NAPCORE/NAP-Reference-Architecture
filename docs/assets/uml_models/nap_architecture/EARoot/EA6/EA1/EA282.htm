<b>Type</b>: Programming Interface<br/><b>Short description</b>: This interface is used for the interaction of the Metadata Requesting System and metadata management module.<br/><b>Contextual Use</b>: MobilityDCAT-AP is a formal metadata specification for mobility data portals as an extension of DCAT-AP. A metadata specification (as any data specification) contains a model with definitions of the syntax and semantics for various data elements, and on how these data elements relate to each other. This way, mobilityDCAT-AP provides precise and unambiguous metadata designations for any mobility-related data offerings, facilitating harmonised, platform-independent metadata descriptions in both human-readable and machine-readable formats.<br/><b>Related Documents</b>: https://napcore.eu/activity-wg2-interoperability-and-level-of-service-of-naps/<br/>=== Communications Requirements ===<br/>As a guideline for minimum expectation the following values can be used<br/><b>Data type</b>: <br/><ul>
<li>Raw data</li></ul>
<b>max bytes per message</b><br/><ul>
<li>up to 500 kBytes per request</li></ul>
<b>maximum delay [s]</b><br/><ul>
<li>system response time below 100ms (independent from user internet connection)  </li><li>delay for submitting request: 0.5s</li></ul>
<b>message interval</b><br/><ul>
<li>On occurrence</li></ul>
<b>security level</b><br/><ul>
<li>level high: authorization by the system, encrypted data transfer on system side</li></ul><p/>
