<b>Type</b>: Programming Interface <br/><b>Short description</b>: Automatic provision of data vie machine to machine interface<br/><b>Contextual Use</b>: The receive or harvest data via machine-to-machine interface aims to harvest existing data in a machine-readable format via request on a broad level from other data platforms or from a single data source.<br/><font color="#ff0000">Prerequisite is the support of the data exchange on both sites via machine-to-machine interface. The interface needs to be registered by the data publisher.</font><br/><font color="#ff0000">Size, frequency, data format, exchange limitations or general specifications are based on the data publication.</font><br/><b>Related Documents</b>: https://napcore.eu/activity-wg2-interoperability-and-level-of-service-of-naps/<br/>=== Communications Requirements ===<br/>As a guideline for minimum expectation the following values can be used<br/><br/><b>Data type</b>: <br/><ul>
<li>Raw data: static, semi-static and dynamic</li></ul>
<b>max bytes per message</b><br/><ul>
<li>static: up to 1 GBytes for data</li><li>semi-static: 10 MByte for incremental updates of data</li><li>dynamic: 1 MByte for dynamic status data</li></ul>
<b>maximum delay [s]</b><br/><ul>
<li>static: system response time below 100ms (independent from user internet connection)</li><li>semi-static: system response time below 10ms (independent from user internet connection)</li><li>dynamic: system response time below 10ms (independent from user internet connection)</li></ul>
<b>message interval</b><br/><ul>
<li>static: one second - assumption that 1 system request</li><li>semi-static: half second - assumption on purpose system request</li><li>dynamic: millisecond - assumption of continuous system request</li></ul>
<b>security level</b><br/><ul>
<li>level high (authorization by the system, encrypted data transfer on system side)</li></ul><p/>
