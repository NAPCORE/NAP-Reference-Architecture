<b>“Security – providers verification”</b> KPI assesses the extent to which the authenticity of provided data (e.g., input provided during the registration process) is verified. In case that a self-verification process is in place for data providers, this KPI takes a value equal to 1. On the other hand, when the authenticity of provided, data is doubled checked by both data providers and NAP operator, this KPI gets a value equal to 2.<br/><br/>Note: This is about metadata of a data provider regarding registration process (i.e., before obtaining username and password or other identity). It’s about trust/credibility of a provider<br/><br/>N/A: No data provider can use the NAP interface<br/>0: No<br/>1: Self-authentication of data providers (e.g., via e-mail)<br/>2: Authentication of data providers is double checked, i.e., by the data provider and approved by the NAP operator<br/><br/>Minimum acceptable value for interoperability: 1<br/>
