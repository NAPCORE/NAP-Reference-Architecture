<b>“Data transfer optimization”</b> KPI assesses the extent to which a NAP can optimize data transferring by compressing the requested content. This KPI is addressed as making sense only if a NAP complies with the concept of a data portal, since a data directory is not involved at all in the data transferring process. A value equal to 0 indicates that such a functionality does not exist, while a value equal to 1 indicates that a NAP serving as a data portal can provide either compressed or uncompressed content based on a client’s request.<br/><br/><br/>Can the NAP server compress the requested content to speed up the transfer?<br/><br/>N/A: Metadata directory case (No data is provided via the NAP online/web interface)<br/>0: No data is provided via the NAP online/web interface in a data platform NAP type or no possibility of data transfer optimization there (i.e., no reaction to a client request, data is provided in an uncompressed form)<br/>0: Data is provided (un)compressed per clients' request<br/>Value+1 (for each of the following options): You can select more options:<br/>a. Data is (or can be) provided compressed (may include compression in Geopackage, WFS/WMS API's, TN-ITS update messages)<br/>b. The user can select if the data will be provided uncompressed or compressed<br/>c. Data can be provided in a differential form, i.e., new data only and/or changes to the previous state where a relevant date/time/period can be set by the user<br/><br/>Minimum acceptable value for interoperability: 1<br/>
