This Function shall be capable of providing the following facilities:<br/><br/>(1) The ability to manage the store of traffic and travel data.<br/>(2) The ability to ensure that all data sent to the store is stored in a coherent and logical manner.<br/>(3) The ability to read, create and overwrite data from the store as and when requested.<br/>(4) The ability to hide data in the NAP Data store  so it could be checked before it is made available. <br/>(5) The ability to carry out its activities in such a way that they do not interfere with one another, and that the integrity of the data being stored and read is preserved. <br/>(6) Ability to send request for Metadata from F3.6.1 Manage NAP Metadata Repository and provide this data to connected functionality, <br/>(7) Ability to store data event logs  <br/><br/>===========<br/><b>CHANGELOG:</b><br/>- change name from “Manage NAP Data” to “F3.6.7 Manage NAP Data Repository“ <br/>- total rewrite of the capability <br/>- this “manage” function now serves as an interface to the datastore for all other functions (dataflows rerouting) <br/>- rerouted dataflows between F3.6.1 and F3.6.10 to F3.6.1. and 3.6.7 and added functionality of F3.6.7 to request and provide metadata for identification of the data against metadata<br/>
