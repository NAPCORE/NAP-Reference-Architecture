<b>Function</b>: An action that processes input data and outputs it to another function or as instructions to an external entity. Functions are divided into High-Level Functions (HLFs), which oversee specific data processing parts and contain Low-Level Functions (LLFs), and LLFs, which handle specific data processing tasks and meet user needs in the FRAME Methodology.<br/><b>Data Store</b>: A repository for data, often managed by LLFs, that serves multiple other LLFs, linking to Functional Areas or HLFs.<br/><b>Data Flow</b>: The movement of data within the system, occurring between LLFs, between LLFs and Data Stores, or between LLFs and external entities.<br/><b>Terminators</b>: External entities not modeled within the FRAME Architecture, such as different vehicle types or traveler roles<br/><b>Metadata directory: </b>The data portal that contains descriptions of the data and services important for their identification, assessment, and subscription by consumer, i.e., metadata. It provides facilities to authenticate data provider and functionality to insert a metadata records. The NAP provides search and discovery services of the hosted records to end users and via machine readable content to other portals. The NAP is not directly involved in the data exchange between data providers and data consumers.<br/>
