Can the NAP server compress the requested content to speed up the transfer?<br/><br/>N/A: Metadata directory case (No data is provided via the NAP online/web interface)<br/>0: No data is provided via the NAP online/web interface in a data platform NAP type or no possibility of data transfer optimization there (i.e., no reaction to a client request, data is provided in an uncompressed form)<br/>0: Data is provided (un)compressed per clients' request<br/>Value+1 (for each of the following options): You can select more options:<br/>a. Data is (or can be) provided compressed (may include compression in Geopackage, WFS/WMS API's, TN-ITS update messages)<br/>b. The user can select if the data will be provided uncompressed or compressed<br/>c. Data can be provided in a differential form, i.e., new data only and/or changes to the previous state where a relevant date/time/period can be set by the user<br/>
