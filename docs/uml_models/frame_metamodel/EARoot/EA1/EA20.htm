Physical Data Flows are the means of communication between Sub-systems, Modules, and Terminators/Actors in the Physical View of ITS. There are two types of Physical Data Flows: Internal, which link Sub-systems and Modules together; and External, which link Sub-systems and Modules to Terminators/Actors. The creation of a Physical View is always based on a particular Functional View, which must have passed the logical consistency checks before it can be used. <br/>
