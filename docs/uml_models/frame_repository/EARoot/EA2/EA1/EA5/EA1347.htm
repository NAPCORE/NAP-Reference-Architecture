Appears In: DFD F5.16 Provide Outputs to Host Vehicle Drivers
This Function shall be capable of providing the following facilities:<br/><br/>(1) The ability to receive the currently valid warnings, lane instructions and traffic regulations for the section of the road network in which the Vehicle is currently located or is about to enter.<br/>(2) A HMI through which the messages to the Driver can be displayed according to the determined priority of messages.<br/>(3) The ability of the HMI to use replicas of the appropriate international roadside signage (according to ISO TS14823:2008) and not "custom" versions of roadside signage in the displays to the Driver.<br/>(4) The ability of the HMI to display the outputs in a way that avoids overloading or distracting the Driver with too much or irrelevant information.<br/>(5) The HMI shall enable the Driver to request output of the most relevant outputs about incidents.<br/>(6) The ability for the HMI to display a message to the Driver saying that a particular service is unavailable, when no data that is relevant to that service has been received for a suitable period of time.<br/>

Details:
(a) continuously monitor for the receipt of any of the input data flows from other functionality that contain information to be output to the Driver;<br/>(b) whenever new content, error or instructions data flows are received in (a), store their contents locally and display them to the Driver through the appropriate output data flow, using replicas of international roadside signs where they exist in a standard way and according to the appropriate standards, e.g. the Vienna convention;<br/>(c) if the number of valid warnings, lane instructions and traffic regulations received in (b) exceeds the maximum amount to be displayed simultaneously, sequence handling shall be executed;<br/>(d) when determining which messages to display in (c). the most messages likely to have the most immediate and serious impact on vehicle and occupant safety shall be displayed first;<br/>(e) if a lane banning instruction is received in a data flow in (a), it must be acknowledged using the lane acknowledgement data flow;<br/>(f) if the possible roadside command & warning error data flow is received in (a) and it contains details of a difference, output the possible command & warning data flow to the Driver;<br/>(g) only continue the output in (f) for as long as the possible roadside command & warning error data flow received in (a) contains details of a difference and remove the output to the Driver in (f) when it does not;<br/>(h) continuously monitor for receipt of the request relevant incident information data flow from the Driver;<br/>(i) when the data flow in (h) is received repeat the output of all incident related messages to the Driver using the appropriate output data flows;<br/>(j) if none of the data flows are received in (a) for a period of time, the display shall show a "system not working" indication to the Driver.<br/>
