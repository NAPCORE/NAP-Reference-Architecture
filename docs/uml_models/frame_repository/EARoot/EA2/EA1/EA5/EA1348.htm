Appears In: DFD F5.16 Provide Outputs to Host Vehicle Drivers
This Function shall be capable of providing the following facilities:<br/><br/>(1) Use its own sensors to obtain details of the commands and warnings that are being output to Drivers from the roadside. <br/>(2) Make comparisons between the details obtained in (1) and the commands and warnings that are being output to Drivers from within the Vehicle.<br/>(3) If the assessment in (2) shows that there is a difference, the Function shall report this to the Provide Commands and dynamic Warnings Function so that a suitable output can be made to the Driver.<br/>

Details:
(a) continuously monitor for receipt of the two input data flows;<br/>(b) when the output commands and warnings data flow is received in (a) store its contents internally;<br/>(c) when the roadside output details data flow is received compare its contents with the data received in (b);<br/>(d) if there a difference is found in (c) then output the possible roadside command & warning data flow to the Output Commands and Dynamic Warnings function;<br/>(e) only output the data flow in (d) if relevant data has been provided by both of the data flows in (b) and (c) so that failure of one form of output does not provide a difference;<br/>(f) once the difference in (d) disappears then send the possible roadside command & warning data flow with no (null) contents.<br/>
