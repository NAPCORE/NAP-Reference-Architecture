This Function shall be capable of providing the following facilities:<br/><br/>(1) The ability to provide a query form or comparable search interface to define keywords or refine demands in different available metadata elements.<br/>(2) The ability to receive and process requests from NAP Content Consumers to be provided with NAP metadata.<br/>(3) The ability to process the request in (1) above so that all suitable search results to the request are retrieved from the Data Store NAP Metadata.<br/>(4) The ability to format the NAP metadata search results obtained in (3) above and send it to the entity that made the request in (2) above or to display them in an overview results page.<br/>(5) Ability to show all metadata at the NAP including data created by the NAP, i.e. quality records in a logical and consistent manner.<br/>(6) The function is providing graphical user interface for performing the said tasks. <br/>(7) Ability to provide all or parts of metadata stored in the NAP Metadata store in a machine-readable format for reasons of harvesting the metadata by Metadata Requesting System (machine to machine access). <br/>(8) The ability to provide all relevant metadata without NAP Content Consumer registration<br/>NOTE: It is assumed that NAP Content Provider could perform all the task of the NAP Content Consumer.<br/><br/>===========<br/>CHANGELOG<br/>- change of meta-data to metadata (also in the function name)<br/>- added NOTE<br/>- addition of functionality 5,6 and 7 and 8 (graphical user interface, showing of existing metadata, providing machine to machine access interface and to get data without registration).<br/>TODO<br/>- the function shall be renamed to Provide NAP Metadata Interface or similar<br/>- check what to do with user registrations, i.e. sometimes they are handy, i.e. for reaching out to NAP Users or for viewing sensitive data (contact information data access links, etc.), <br/><br/>
