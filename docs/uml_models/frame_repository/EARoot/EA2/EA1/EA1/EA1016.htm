This Function shall be capable of providing the following facilities:<br/><br/>(1) A HMI through which the Traveller or Driver can be guided to provide the precise definition of the service that they want, and the way that they wish to pay for it.<br/>(2) The HMI shall also enable the Traveller or Driver to select the appropriate contract, inform them about their rights to use the service, and display the associated price.<br/>(3) The ability for the Function to be activated by functionality in the Provide Traveller Journey Assistance Area, or even directly for simple operations.<br/>(4) The ability to ask for a credit check to be done on the Traveller or Driver and also to check to see whether an advance payment has been made.<br/>(5) The ability to initiate the recovery of any advanced payment, or else initiate the computation of the fee.<br/>

(a) get the information about the service requested from the Provide Traveller Journey Assistance Functional Areas, or directly from the user;<br/>(b) if this Functional Area just asks for a tariff (and not a transaction), go to (g);<br/>(c) get user's ID;<br/>(d) obtain the corresponding contracts;<br/>(e) get user's choice of contract;<br/>(f) check with "Access" and/or "Credit Control" and, if necessary, register an incorrect payment (fraud);<br/>(g) send the choice and contract to the Check Advanced Payment function to check if advanced payment has been made;<br/>(h) send the different elements to the Compute Service Fee function to initiate the computation of the corresponding fee;<br/>(i) if payment has already been completely made, send an agreement to the Access Control function;<br/>(j) if not, or only part payment has been made, send the elements to the Recover Fee function to recover any fees.<br/>
