This Function shall be capable of providing the following facilities:<br/><br/>(1) The ability to collect and send for storage data (e.g. location, status, alarms, occupancy) in real-time from PT Vehicles through any suitable data communications links.<br/>(2) When fault data is received from PT Vehicles, the ability to pass it on to maintenance functionality.<br/>(3) The ability to provide PT Vehicle data, include locations that are combined with digital map data, to the PT Operator interface functionality.<br/>(4) The ability to exchange with PT Drivers in PT Vehicles voice and data messages concerning unusual situations that are not included in updates provided by regularly transmitted data, including such things as emergencies, special messages to PT Drivers, etc.<br/>

Details:
(a) collect, filter and if necessary fuse all the data received from the Monitor PT Vehicles function;<br/>(b) collect map data from the Geographic Map Data provider so that the PT Vehicle positions can be determined on a map;<br/>(c) periodically send the processed data from (a) to the stores of Historic PT Vehicle Data and Real Time PT Vehicle Status Data and to the Provide Operator Interface for Manage PT Fleet function in the historical vehicle information data flow;<br/>(d) if any of the data flows in (a) indicate that there is a possible fault with the PT Vehicle then send it to the Monitor PT Infrastructure function in the possible PT Vehicle fault data flow;<br/>(e) if the message from driver data flow arrives from the Monitor PT Vehicles function pass its contents to the Provide Operator Interface for Manage PT Fleet function in the communications from driver data flow;<br/>(f) if the data flow communications to driver is received, send it to the Monitor PT Vehicles function in the message to driver data flow.<br/>
