This Function shall be capable of providing the following facilities:<br/><br/>(1) Support for both data and voice dialog between the PT Driver and the Collect PT Vehicle Data functionality through a suitable HMI for use in situations that are not included in updates provided by regularly transmitted data, e.g. emergencies, special messages to drivers, etc.<br/>(2) The output of other PT Vehicle data and alarm information to the PT Driver through a suitable HMI.<br/>(3) The ability of the PT Driver to use a suitable HMI to request information about the service (schedule and fares) on which the PT Vehicle is to be used and for that information to be displayed.<br/>

Details:
(a) continuously monitor for the arrival of input data flows from PT Drivers;<br/>(b) send messages from the PT Driver to the Collect PT Data function in the message from driver data flows<br/>(c) continuously monitor for the arrival of the message to driver data flow;<br/>(d) when the data flow in (c) arrives immediately output its contents to the PT Driver;<br/>(e) continuously monitor for the receipt of the assign vehicle to service data flow from the PT Driver;<br/>(f) when the data flow in (e) arrives, put the identity of the service that it contains into the request schedule for vehicle and request fares for vehicle data flows and send them to the Plan New Service Schedule and Manage Fare Schemes Data Store functions;<br/>(g) as a result of (f) continuously monitor for the receipt of the requested service for vehicle and requested fares for vehicle data flows<br/>(h) when the data flows in (g) arrives, store the data locally within the function.<br/>
