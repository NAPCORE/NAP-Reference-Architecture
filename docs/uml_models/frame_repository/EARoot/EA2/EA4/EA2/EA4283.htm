This Generic Terminator Role Content Consumer shall represent any interested entity in traffic, travel data. The NAP Content Consumer can find available metadata via the NAP. Therefore, Function F3.6.4 Provide Access for NAP Metadata Retrieval offers an interface (typically a graphical user interface freely accessible via the Internet) to search the metadata in D3.15 NAP Metadata.<br/>The NAP Content Consumer also can, in case of Data Platform NAP type retrieve data directly from the NAP, via Provide data provision interface function. In some cases, especially when the data are pushed to the Content Consumer systems and / or authorisation is necessary the Content Provider must register at the NAP to obtain credentials and / or to set up a machine account to which data are pushed. <br/><b>Key issue</b>: Request high quality services; provide feedback about relevant content.<br/><b>Area of responsibility</b>: Be compliant to the agreement with content provider<br/><b>Tasks</b>: Receive content <br/><b>Mnemonic</b>: nap.cc<br/><br/>===========<br/><b>CHANGELOG:</b><br/>- added possibility to register machine account and to register CC and to obtain credentials (last paragraph)<br/><b>TODO:</b><br/>- check data flows so they contain the correct information<br/><br/>
