This Terminator shall represent the organisation(s) that process all electronic fund payment requests originated from functionality in the system.  Typically these payment requests shall be from Travellers for the provider of a service such as trip planning, parking, tolls or to top up a PT Fare Card.  It shall be possible for functionality in the system to send data objects containing details of the payment, include the originator and the receiver to this Terminator.  In response it is expected that this Terminator shall provide data objects that show either that the payment request was successful or that it has failed, in which case the data objects may include a reason.  The functionality in the system must also be able to communicate with more than one organisation that processes electronic fund payment requests.<br/><br/>Mnemonic: fc<br/>
