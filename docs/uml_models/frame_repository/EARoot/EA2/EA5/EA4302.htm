This Data Store shall be used within the Manage Traffic Functional Area.  It shall contain information objects that provide metadata about traffic, travel data and any other relevant information. The required and optional elements of metadata have been harmonised and can be found in a metadata catalogue: https://mobilitydcat-ap.github.io/mobilityDCAT-AP/releases/index.html.<br/>Metadata is structured in the data store to:<br/>(1) describe data provider, where each data provider may list multiple of <br/>(2) data sets or services and each of such can be provided by more than one way by specifying its<br/>(3) distribution parameters. <br/>The content and the structure are governed by the mobilityDCAT-AP.<br/>Further on the data store also contains:<br/>(4) log of events with respect of metadata creation, update and <br/>(5) quality information per data set.<br/><br/>==========<br/><b>CHANGELOG:</b><br/>- changed meta-data to metadata<br/>- deleted specific information about some metadata components<br/>- added information about structure of the catalogue and additional data stored within the data store<br/>- deleted specific information about data types that can be shared<br/><b>TODO:</b><br/>- decide what detail of mobilityDCAT-AP to include<br/>- IMPORTANT: investigate where to log in data and user events, where to hold information about active subscription<br/>- IMPORTANT: decide how to connect metadata data base and data database<br/><br/>
