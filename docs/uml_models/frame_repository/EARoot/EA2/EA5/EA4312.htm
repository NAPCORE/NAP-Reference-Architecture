This Data Store shall be used within the Manage Public Transport Operations Functional Area.  It shall contain information objects that provide the historical average value indicators for Public Transport vehicles in the fleet.<br/><br/>The information objects in the Store shall contain the following data objects that provide the static data about each PT Vehicle:<br/>(1) Type (characters),<br/>(2) Vehicle ID (number),<br/>(3) Passenger capacity (number).<br/><br/>The information objects in the Store shall contain the following data objects for the real-time data about each PT Vehicle:<br/>(4) Date (date string),<br/>(5) Time (numbers defining hours, minutes and seconds),<br/>(6) Location (characters and/or numbers, e.g. GPS/Galileo data),<br/>(7) Service ID (number),<br/>(8) Real-time status indicators (numbers and/or characters),<br/>(9) Alarm present (number or character indicating YES or NO),<br/>(10) Number of passengers currently on-board (number),<br/>(11) Passenger alarm raised (number or character indicating YES or NO),<br/>(12) Passenger alarm acknowledged (number or character indicating YES or NO),<br/>(13) Message sent to driver (number or character indicating YES or NO),<br/>(14) Message text sent to driver (text string),<br/>(15) Message received from driver (number or character indicating YES or NO),<br/>(16) Message text received from driver (text string),<br/>(17) Internal video from PT Vehicle (video file),<br/>(18) Scheduling for PT services.<br/><br/>It shall be possible for the data objects in each information object to be stored in such a way that they can be retrieved by route, by service, by type of PT Vehicle etc. as needed.  For each of the above categories as a minimum the stored data objects shall be archived and classified by type of day with the needed level of detail (working days, holidays, school days, off school days, rainy days, sunny days, traffic restriction days, etc.).  The level of detail shall be coherent with the detailed specification of the system: e.g. average travel time can be as detailed as calculated for every few meters of road along the route or distinguishing between characteristic stretches of road, or just between two terminals.<br/><br/>The information objects in the Store shall also include the exact description of the time, place and whatever else is needed to identify the scenario to which each is expected to be applied.<br/><br/>Stored data shall be constantly updated with new measurements available and previously smoothed and validated.  It shall be possible for the time slots used to store the data to vary according to the needs.<br/><br/>The information objects that are archived shall also be filtered taking into account presence of exceptional events during the day and the actual operating service scheme.<br/>
