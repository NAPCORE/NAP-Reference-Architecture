This Data Store shall be used within the Provide Electronic Payments Facilities Functional Area.  It shall contain information objects that provide all the characteristics of the services that are available to the users (Travellers), including the nature of those services and the identities of the organisations that are providing them.<br/><br/>The information objects for each service shall contain the following data objects:<br/>(1) Service ID (number),<br/>(2) Nature of service (characters),<br/>(3) ID of organisation providing the service (number),<br/>(4) Associated account, i.e. where the payment will go (characters and/or numbers),<br/>(5) Location of service, i.e. where the user can use it (characters, e.g. nation, region, city, state, area),<br/>(6) Types of contracts possible (characters),<br/>(7) Categories of people allowed to use this service (that is to pass a contract for it, regardless of access rights which are defined by regulating bodies) (characters),<br/>(8) Enforcement procedures (characters),<br/>(9) Modes of booking (characters),<br/>(10) Identification of tariffs (pointer to tariff data store) (numbers),<br/>(11) Rules of fee apportionment if several operators provide the same service (characters),<br/>(12) List of the ID's of services grouped for the apportionment (numbers).<br/><br/>The information objects in the Store shall be organised so that data for each service is kept together.<br/>
