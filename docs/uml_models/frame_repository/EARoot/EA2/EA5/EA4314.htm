This Data Store shall be used within the Manage Public Transport Operations Functional Area.  It shall contain information objects that provide data about the inter-urban and urban road networks for use by the Manage Public Transport Area.  The data in the Store will be used to enable new or revised services and schedules to be planned.<br/><br/>The information objects in the Store shall contain the following data objects:<br/>(1) Road type (numbers and characters),<br/>(2) Type of data (numbers and characters),<br/>(3) Link:<br/>      (3a) Link ID (number),<br/>      (3b) Type (characters),<br/>      (3c) Start location (characters and/or numbers, e.g. GPS/Galileo data),<br/>      (3d) End location (characters and/or numbers, e.g. GPS/Galileo data),<br/>      (3e) Length (number),<br/>      (3f) Bus lane present indicator (number or character indicating YES or NO),<br/>      (3g) Vehicle type usage restrictions (numbers and characters),<br/>      (3h) Speed limit(s) (numbers in kph or mph),<br/>      (3i) Presence of other objects, e.g. tunnels/bridges (numbers and characters),<br/>      (3j) Obstructions, e.g. narrow road/lanes, bridges/tunnels giving low clearance, bridges with weight restrictions (characters),<br/>      (3k) Stop present (number or character indicating YES or NO),<br/>      (3l) Stop ID (numbers),<br/>      (3m) Stop location (characters and/or numbers, e.g. GPS/Galileo data),<br/>(4) Junction:<br/>     (4a) Junction ID (number),<br/>     (4b) Type (characters),<br/>     (4c) Location (characters and/or numbers, e.g. GPS/Galileo data),<br/>     (4d) Equipment installed (numbers and characters),<br/>            (4d1) Configuration (phases/durations/turning movements): (numbers and characters),<br/>            (4d2) Available methods of management (numbers and characters),<br/>            (4d3) Non-available vehicle movements (numbers and characters),<br/>            (4d4) PT Vehicle priority available (number or character indicating YES or NO),<br/>             (4d5) Next up-stream junction (numbers and characters),<br/>            (4d6) Next down-stream junction (numbers and characters),<br/>(5) Date of last update (date string),<br/>(6) Access details, e.g. stairs, lifts, ramps, narrow doors, disabled toilets, baby changing facilities and waiting rooms (numbers and characters),<br/>(7) Local special locations served by stop, e.g. tourist attraction, sports stadium, work place and entertainment venue, etc. (numbers and characters),<br/>(8) Other information (characters).<br/><br/>There shall be one of these Data Stores for each of the road networks within which the functionality in the Manage Public Transport Operations Functional Area is implemented.  The data objects in each of the Stores shall be provided by functionality from the Manage Traffic Functional Area.  It shall be possible for the data objects to be provided, amended and enhanced by the PT Operator if no functionality from the Manage Traffic Functional Area is available to provide them.<br/>
