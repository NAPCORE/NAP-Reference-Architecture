Appears In: DFD F5.14 Provide In-vehicle Trip Planning & Implementation
This Data Store shall be used within the Provide In-vehicle Trip Planning High-level Function.  It shall contain information objects that are the result of the process to create a Vehicle Trip Plan.<br/><br/>The information objects in the Store shall contain the following objects:<br/>(1) Route ID (number),<br/>(2) Route name (characters),<br/>(3) Origin (characters and/or numbers, e.g. GPS/Galileo data),<br/>(4) Destination (characters and/or numbers, e.g. GPS/Galileo data),<br/>(5) Expected arrival time (numbers defining hours, minutes and seconds),<br/>(6) Vehicle type for route (characters),<br/>(7) Type of goods carried, if any (characters),<br/>(8) Route (data for a digital roadmap using a standard format, e.g. GDF, plus characters and/or numbers, e.g. GPS/Galileo data),<br/>(9) Rest zone:<br/>    (9a) Rest zone ID (number),<br/>    (9b) Name (characters),<br/>    (9c) Location (characters and/or numbers, e.g. GPS/Galileo data),<br/>    (9d) Booked start date (date string),<br/>    (9e) Booked start time (numbers defining hours, minutes and seconds),<br/>    (9f) Booked stay duration numbers defining hours, minutes and seconds),<br/>(10) Un/loading zone:<br/>    (10a) Un/loading zone ID (number),<br/>    (10b) Name (characters),<br/>    (10c) Location (characters and/or numbers, e.g. GPS/Galileo data),<br/>    (10d) Booked start date (date string),<br/>    (10e) Booked start time (numbers defining hours, minutes and seconds),<br/>    (10f) Booked stay duration (numbers defining hours, minutes and seconds),<br/>(11) Payments made:<br/>    (11a) What for (characters)?<br/>    (11b) Complete (number or character indicating YES or NO),<br/>(12) Oher relevant data (characters and/or numbers).<br/><br/>It shall be possible for each information object to include data objects about more than one rest area and/or un/loading zone and for payments for use to be recorded against some or all of these, plus any pre-payment of road use charges.  The data objects in this Store shall be retained for the prime purpose of supporting the Diver during the implementation of the Trip Plan.  All the factors that were considered in production of the Trip Plan are included in the data objects described above.<br/>
