This Data Store shall be used within the Provide Traveller Journey Assistance Functional Area (F6).  It shall contain data that is the result of the trip planning process.  This data is retained for the prime purpose of supporting the Traveller during the trip. The most notable requirement is to react to the consequences of perturbations in the situation(s) existing during trip planning.  All the considerations that resulted in production of the trip itinerary are included in this Data Store.<br/><br/>The data in the Store shall be structured in the following way:<br/>(1) Traveller s ID (number),<br/>(2) Traveller's preferred language (characters),<br/>(3) Traveller's disabilities (characters),<br/>(4) Route (data for a digital roadmap using a standard format, e.g. GDF, plus characters and/or numbers, e.g. GPS/Galileo data),<br/>(5) Trip origin (characters and/or numbers, e.g. GPS/Galileo data),<br/>(6) Departure date (date string),<br/>(7) Departure time (numbers defining hours, minutes and seconds),<br/>(8) Way point:<br/>    (8a) Location (characters and/or numbers, e.g. GPS/Galileo data),<br/>    (8b) Expected arrival date (date string),<br/>    (8c) Expected arrival time (numbers defining hours, minutes and seconds),<br/>    (8d) Car park information (characters),<br/>(9) Trip destination (characters and/or numbers, e.g. GPS/Galileo data),<br/>(10 Expected arrival date (date string),<br/>(11) Expected arrival time (numbers defining hours, minutes and seconds),<br/>(12) Car park information (characters),<br/>(13) Description of cargo to be carried on trip (characters),<br/>(14) Number of passengers expected in Vehicle (number),<br/>(15) Hotel bookings completed (characters),<br/>(16) Hotel bookings still to be made (characters),<br/>(17) Car rental bookings completed (characters),<br/>(18) Car rental bookings still to be made (characters),<br/>(19) Other travel mode bookings completed (characters),<br/>(20) Other travel mode bookings still to be made (characters),<br/>(21) Vehicle breakdown assistance information (characters),<br/>(22) Way of communication with Traveller while on-trip (characters + numbers),<br/>(23) Route guidance facilitation parameters (characters),<br/>(24) Trip skeletons (see below).<br/><br/>It shall be possible for there to be none, or more than one way point, plus hotel, car rental and other travel mode bookings.<br/><br/>A special feature of the data in the Store to support the Traveller shall be the "trip skeleton", which shall be available for use in two ways:<br/>(a) as the start of the search for alternatives during trip planning,<br/>(b) as the fixed part of the trip if the trip has to be retained for future reuse.<br/>The core part of the "skeleton" shall be the origin-destination combination and a date and/or time for the start or the arrival and possibly a number of intermediate locations - see data to be stored above.  In both cases, trip preferences are part of the trip skeleton for trip planning purposes and it shall be possible to obtain these from the store of GTP data.<br/><br/>The data for each Traveller must be kept separate with no possibility of inadvertent access to the data for one Traveller by either another Traveller, or some form of un-authorised access.<br/>
