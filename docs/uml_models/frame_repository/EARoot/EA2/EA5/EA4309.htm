Appears In: DFD F3.1.2 Provide Inter-urban Traffic Management
This Data Store shall be used within the Manage Traffic Functional Area.  It shall contain information objects that provide the static data for the inter-urban traffic road network managed by the System.  The static data shall cover the actual layout and configuration of the inter-urban road network.<br/><br/>Each information object in the Store shall contain the following data objects:<br/>(1) Road type: (numbers and characters),<br/>(2) Type of data: (numbers and characters),<br/>(3) Link:<br/>     (3q) ID (number),<br/>     (3b) Type (characters),<br/>     (3c) Start location (characters and/or numbers, e.g. GPS/Galileo data),<br/>     (3d) End location (characters and/or numbers, e.g. GPS/Galileo data),<br/>     (3e) Length: (number),<br/>     (3f) Number of lanes/carriageways (number),<br/>     (3g) Lane/carriageway width(s) (number(s)),<br/>     (3h) Vehicle type usage restrictions (numbers and characters),<br/>     (3i) Vehicle parking restrictions (numbers and characters),<br/>     (3j) Vehicle speed limit(s) (number(s) in kph or mph),<br/>     (3k) Bus lane present indicator (number or character indicating YES or NO)<br/>      (3l) Presence of other objects, e.g. tunnels/bridges (numbers and characters)<br/>     (3m) Obstructions, e.g. narrow road/lanes, bridges/tunnels giving low clearance, bridges with weight restrictions (characters)<br/>(4) Date of last update (date string).<br/><br/>These data objects shall be used by a variety of functions to enable traffic in the inter-urban network to be managed.  It shall also be provided to functionality in the Manage Public Transport Operations Functional Area so that its service routes and schedules can be planned.  It shall be possible for some data objects to also be provided to Vehicles to enable driver guidance and information to be provided.<br/>
