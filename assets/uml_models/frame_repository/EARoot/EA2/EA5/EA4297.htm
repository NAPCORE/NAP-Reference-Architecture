Appears In: DFD F3.1.5 Provide Management of Service Areas
This Data Store shall be used within the Manage Traffic Functional Area. It shall contain information objects that provide the static data for the service areas that are accessed from the inter-urban traffic road network managed by the System. <br/><br/>Each information object in the Store shall contain the following data objects:<br/>(1) Service area location (characters, e.g. nation, region, city, state, area names(s), but may also include characters and/or numbers, e.g. GPS/Galileo data)<br/>(2) Inter-urban road identity if known (characters and number)<br/>(3) Number of entrances (number)<br/>(4) Number of exits (number)<br/>(5) Total number of spaces (number)<br/>(6) Float to take account of vehicles searching for spaces (numbers)<br/>(7) Owner/operator ID (number) <br/>(8) Owner/operator contact information (characters and/or numbers)<br/>(9) Charges (numbers - in currency)<br/>(10) Periods when they operate (time and date string)<br/>(11) Method of service area fee collection (characters)<br/>(12) Hours when operator present (time and date string)<br/>(13) Separate motorcycle spaces (number)<br/>(14) Number of disabled spaces (number)<br/>(15) Number of spaces for coach operators (number)<br/>(16) Number of spaces for HGV's (number)<br/>(17) Re-fuelling facility available (character indicating YES or NO)<br/>(18) Includes hotel(s) with number (character indicating YES or NO plus number)<br/>(19) Includes restaurant(s with number (character indicating YES or NO plus number)<br/>(20) Includes shop(s) with number (character indicating YES or NO plus number)<br/>(21) Display information in urban area (character indicating YES or NO)<br/>(22) List of devices on which service area information is displayed with details about whether "state" or "spaces" is to be displayed (characters and/or numbers)<br/>(23) Thresholds for service area status change, e.g. number of occupied spaces for "almost full" status to be achieved (characters + numbers)<br/><br/>It shall be possible for the service area state calculation Function to calculate the current state of the service area using the following "live" data:<br/><br/>(24) Number of vehicles entered service area (number)<br/>(25) Number of vehicles that have left the service area (number)<br/>(26) Service area status, e.g. "almost full", "full", "spaces" and "closed" (characters)<br/><br/>It shall be possible for this data to be replicated depending on the number of entrances and exits that a service area has and how the data that they collect is to be used.  For example it shall be possible for the service area to have separate parking areas for different types of vehicle such as car, coaches and HGV's.<br/>
