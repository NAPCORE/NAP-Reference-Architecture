<b>Type</b>: User interface <br/><b>Short description</b>: This interface is used to provide data to the NAP by content provider.<br/><b>Contextual Use</b>: !!todo<br/><b>Related Documents</b>: https://napcore.eu/activity-wg2-interoperability-and-level-of-service-of-naps/<br/>=== Communications Requirements ===<br/>As a guideline for minimum expectation the following values can be used<br/><b>Data type</b>: <br/><ul>
<li>static data (Raw data, binary,...)</li></ul>
<b>max bytes per message</b><br/><ul>
<li>up to 1 GBytes for attached files (hard to define a minimum)</li></ul>
<b>maximum delay [s]</b><br/><ul>
<li>system response time below 100ms (independent from user internet connection)  </li><li>delay for uploading attached files: 10s per MB</li></ul>
<b>message interval</b><br/><ul>
<li>one minute - assumption that two data provider are putting data in parallel at the same time; depends on client numbers</li></ul>
<b>security level</b><br/><ul>
<li>level high (authorization by the system, encrypted data transfer on system side)</li></ul><p/>
