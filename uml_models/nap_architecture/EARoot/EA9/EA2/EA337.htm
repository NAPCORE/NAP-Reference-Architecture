The first feature of the data discovery subset involves the <b>traceability of data hosted by/made available through NAPs</b>. This feature is associated with the <b>availability of discovery services and searching functionalities</b>, such as engines and search masks supporting querying and filtering.<br/>Another feature classified in the discoverability subset involves the <b>provision by a NAP of the appropriate metadata</b> enabling the exploitation of (any available) discovery services. It should be noted that metadata is an important aspect for the reusability and discoverability of data resources, providing additional information that helps data consumers better understand the meaning of data, its structure, and to clarify other issues, such as usage rights and license terms. If insufficient metadata are provided, the actual data resources would not be easily discoverable and, thus, exploitable by parties other than the data providers. <br/>Another important, not yet mandatory, feature is the <b>machine readability of metadata enabled</b> through exportability using certain formats (e.g., XML and RDF DCAT-AP streams). In principle, it should be bear in mind that metadata should facilitate the discovery of data by both human and machines.<br/>Finally, the last feature included in that subset, which can also be addressed as non-mandatory, constitutes <b>the support for map-based search of datasets</b>. Web maps can provide a more advanced way of searching, allowing NAP users to search at national, regional and local level, for data elements and resources.<br/><br/>
