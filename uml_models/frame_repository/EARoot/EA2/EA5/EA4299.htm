This Data Store shall be used in the Manage Traffic Functional Area.  It shall contain information objects that provide details of incident strategies that have been created and used previously.<br/><br/>Each information object in the Store shall contain the following data objects:<br/>(1) Strategy identity (characters)<br/>(2) Reason for implementing the strategy (characters)<br/>(3) How the strategy was implemented, e.g. Operator, Automatically - (character);<br/>(4) Implementation date (date string)<br/>(5) Implementation time (numbers defining hours, minutes and seconds)<br/>(6) Removal date (date string)<br/>(7) Removal time (numbers defining hours, minutes and seconds)<br/>(8) Part(s) of the road network covered by the strategy (data for a digital roadmap using a standard format, e.g. GDF)<br/>(9) Set of actions for the urban road network traffic management functionality - optional (characters);<br/>(10) Set of actions for the inter-urban road network traffic management functionality - optional (characters);<br/>(11) Set of other actions - optional (characters);<br/>(12) Set of text messages to be displayed to Travellers and/or Drivers, with the identities of the equipment through which the displays shall be shown (characters).<br/><br/>The above shall represent the "minimum" set of information about each strategy.  It shall be possible for it to be supplemented by other sets of information such as comments from the Road Network Operator.  Some of the data items shall only be populated when the strategy is used, e.g. how the strategy was implemented, date/time of implementation/removal.<br/><br/>It shall be possible for the actions to be a series of commands that can be sent directly to other Functions, such as those concerned with traffic management.  Other actions shall be able to be requests for the Road Network Operator to call one or more Emergency Services, or take other action that cannot be implemented by a specific Function.<br/><br/><br/>
