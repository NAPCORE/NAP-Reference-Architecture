This Function shall be capable of providing the following facilities:<br/><br/>(1) The ability to translate service area occupancy levels into service area "states".<br/>(2) The ability for the translation to enable the information output functionality to show either spaces or "state" according to the type of equipment that is available.<br/>(3) The ability for the occupancy levels to be provided by the service area occupancy monitoring functionality.<br/>

(a) when the trigger output data flow is received, the data it contains shall be translated into a service area occupancy and status;<br/>(b) when (a) is completed, send all the data flows containing service area occupancy to other parts of the Manage Traffic Functional Area and to the Manage Public Transport Operations Functional Area and load the data into the store of Inter-urban Service Area Data using service area status and occupancy data flows;<br/>(c) in addition to (b) send the service area occupancy information and status data flows to the Output Service Area Messages and Provide Operator Interface functions;<br/>(d) if the request service area details data flow is received from the Manage Public Transport Operations Functional Area, collect the required data from the store of Inter-urban Service Area Data using the data flow for service area static data;<br/>(e) if the override service area status message is received, repeat (b) and (c) with the new status being imposed by the operator.<br/>
